# 🔐 认证系统使用指南

## ❓ 关于登录账号和密码

**重要说明**: 这个项目使用 Supabase 认证系统，**没有预设的登录账号**。您需要自己注册账号才能使用。

## 🚀 快速开始 - 创建第一个账号

### 步骤 1: 访问登录页面
1. 打开您的部署网站
2. 访问 `/login` 路径，或者主页会自动重定向

### 步骤 2: 注册新账号
1. 在登录表单中输入：
   - **邮箱**: 使用真实邮箱（如需验证）或测试邮箱
   - **密码**: 建议 6 个字符以上
2. 点击 **"Sign Up"** 按钮进行注册

### 步骤 3: 确认注册
- 如果启用了邮箱验证：检查邮箱并点击确认链接
- 如果未启用验证：直接可以登录

### 步骤 4: 登录使用
1. 使用刚注册的邮箱和密码
2. 点击 **"Sign In"** 按钮
3. 登录成功后进入主页

## 🔧 认证系统功能

### ✅ 当前支持的功能
- 📝 **用户注册** - 邮箱 + 密码
- 🔐 **用户登录** - 安全认证
- 🚪 **自动登出** - 点击登出按钮
- 🛡️ **路由保护** - 未登录自动重定向
- 💾 **会话管理** - 自动保持登录状态

### 🎯 用户体验流程
1. **首次访问** → 重定向到登录页面
2. **注册账号** → 创建新用户
3. **登录成功** → 进入受保护的主页
4. **浏览应用** → 正常使用所有功能
5. **登出** → 清除会话，返回登录页面

## 🌐 页面说明

### 🏠 主页 (/)
- **未配置 Supabase**: 显示配置指导界面
- **已配置但未登录**: 重定向到登录页面
- **已登录**: 显示用户信息和项目功能

### 🔐 登录页面 (/login)
- **注册功能**: Sign Up 按钮
- **登录功能**: Sign In 按钮
- **状态提示**: 显示配置和连接状态
- **错误处理**: 友好的错误提示

## 🔍 故障排除

### 问题 1: 无法注册或登录
**可能原因**:
- Supabase 环境变量未配置
- Supabase 项目认证未启用
- 网络连接问题

**解决方案**:
1. 检查 Vercel 环境变量配置
2. 确认 Supabase 项目状态
3. 查看浏览器控制台错误信息

### 问题 2: 注册后无法登录
**可能原因**:
- 邮箱验证未完成
- 密码不符合要求
- 用户被禁用

**解决方案**:
1. 检查邮箱验证邮件
2. 确认密码长度和复杂度
3. 在 Supabase Dashboard 检查用户状态

### 问题 3: 页面显示配置提示
**原因**: Supabase 环境变量未正确配置

**解决方案**:
1. 在 Vercel Dashboard 设置环境变量
2. 确保变量名称正确
3. 重新部署应用

## 🔑 Supabase 后台管理

### 查看用户列表
1. 访问 [Supabase Dashboard](https://supabase.com/dashboard)
2. 选择您的项目
3. 导航到 `Authentication` → `Users`
4. 查看所有注册用户

### 手动创建用户
1. 在用户列表页面点击 `Add user`
2. 输入邮箱和密码
3. 设置用户角色（可选）
4. 创建用户

### 用户管理操作
- 🔄 **重置密码**: 为用户重置密码
- 🚫 **禁用用户**: 临时禁用账号
- 📧 **发送验证邮件**: 重新发送验证
- 🗑️ **删除用户**: 永久删除账号
